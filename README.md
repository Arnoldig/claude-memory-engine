# claude-memory-engine

Переиспользуемый «движок памяти уроков» для Claude Code. Ставится на любую машину одной командой и даёт проекту самообслуживаемую систему накопления и подсказки уроков. (*A reusable lessons-memory engine for Claude Code — install once, get self-maintaining lesson capture & retrieval.*)

Извлечён из инструментария проекта ЧеКи (#memory-tooling-library). В библиотеку входит ТОЛЬКО механизм; сами уроки конкретного проекта (бизнес-знания, приватные данные) в неё не входят и хранятся отдельно.

## Зачем это нужно (простыми словами)

Когда работаешь с ассистентом над проектом долго, накапливаются «уроки»: как тут принято, на какие грабли уже наступали, что нельзя ломать. Если хранить их как попало, ассистент про них забывает, а список разрастается и устаревает. Этот движок превращает уроки в аккуратную, самоподдерживающуюся систему: нужный урок сам всплывает в нужный момент, указатель собирается автоматически, старое архивируется, а параллельные сессии не затирают правки друг друга.

## Что внутри (8 механизмов и их польза)

1. **Хранилище уроков** — маленькие markdown-файлы с frontmatter (`name`, `description`, `topic`, `keywords`, `applies_to`) + одно «горячее ядро» с лимитом размера. *Польза: знания структурированы и не превращаются в свалку.*
2. **Авто-указатель CATALOG** (`catalog_generate`) — индекс собирается из поля `topic:` каждого урока, руками его не ведут. *Польза: список уроков всегда полон и не дрейфует.*
3. **Офлайн-ретривер** (`memory_retrieve`) — на каждый запрос без обращения к ИИ (ноль токенов) ранжирует уроки по словам и подсказывает релевантные, молча если ничего важного. *Польза: нужный урок всплывает сам, бесплатно.*
4. **Уроки по пути файла** (`applies_to`) — урок с глобом `applies_to` показывается перед первой правкой совпавшего файла. *Польза: предупреждение приходит ровно там, где собираешься менять код.*
5. **Авто-обслуживание** (`memory_archive`, `precedent_index`) — старые записи уезжают в архив, большие архивы становятся адресуемыми (не нужно читать целиком). *Польза: память не раздувается и не забивает рабочий контекст.*
6. **Защита параллельных сессий** (`memory_concurrency`) — если две сессии правят один файл памяти, вторая получает мягкий отказ «перечитай и повтори» вместо тихой потери. *Польза: правки не пропадают.*
7. **Страж выбора модели суб-агентов** (`subagent_model_guard`) — не даёт запустить «рутинного» помощника на самой дорогой модели по забывчивости. *Польза: экономия без потери качества.*
8. **Замер делегирования** (`subagent_efficiency_log`) — пишет журнал запусков помощников, чтобы видеть, оправдана ли схема. *Польза: решения о делегировании — на фактах, не на догадках.*

## Архитектура (три слоя)

- **Логика** — пакет `claude_memory/` (чистый Python, ноль зависимостей): 8 модулей + `config.py` (вся параметризация) + `applies_to.py` + `hooks_cli.py` (единый диспетчер хук-логики).
- **Обёртки** — `hooks/cme_hook.sh`: одна тонкая обёртка, которую Claude Code дёргает на события (`bash cme_hook.sh <событие>`); вся логика — в `hooks_cli`.
- **Данные** — уроки конкретного проекта (в движок НЕ входят, приватны).

Никаких путей конкретной машины в коде нет: всё берётся из конфига (`MemoryConfig`) и из расположения самих файлов.

## Установка (вариант A — git + install.sh)

```
git clone <repo-url> claude-memory-engine
cd claude-memory-engine
./install.sh /path/to/your/project /path/to/your/memory
```

install.sh: кладёт движок в `<project>/.claude/memory_engine/`, ставит обёртку в `<project>/.claude/hooks/`, создаёт конфиг `<project>/.claude/claude-memory.config.json` и **дописывает** регистрацию хуков в `settings.json`, не затирая чужие. Повторный запуск идемпотентен (дубли не плодятся). Хуки активируются со следующей сессии Claude Code.

Аргументы по умолчанию: проект — текущий каталог, память — `~/.claude/memory`.

## Конфигурация

Все настройки — в `<project>/.claude/claude-memory.config.json`. install.sh пишет туда минимум (пути проекта); остальное берёт из дефолтов. Полный список опций с пояснениями — `examples/claude-memory.config.json` (таксономия тем, имена особых файлов, пороги размеров, рутинные типы суб-агентов, подстрока «самой сильной модели», окна авто-архива, и т. д.). Дефолты нейтральны (общая таксономия, английские заголовки); под свой проект и язык — перекройте в конфиге.
